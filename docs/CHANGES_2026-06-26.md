# Changelog — 2026-06-26

A large session: a new Asana import/staging/distribute system, an Asana-like subtask
model, a full UI/UX overhaul, and the actual data import of all active Asana projects.
Everything below is **deployed to production** (`./deploy.sh`, verified up).

## New feature: Asana → Staging → Distribute
Full details in **[ASANA_STAGING.md](ASANA_STAGING.md)**. In short:
- Import Asana projects into a **hidden, admin-only Staging area** (invisible to workers
  in the projects list, dashboard, search, and direct URLs).
- Admin **copies** staged tasks/subtasks into real projects (existing or new) for people;
  the staged original stays and turns green ("Copied ✓"). The **original Asana owner** is
  preserved and shown.
- **"✨ Make into task"**: a staged subtask is expanded by AI into a full task (description
  + subtasks) for whoever you assign.
- New: `POST /api/admin/import` (idempotent, secret-guarded), `GET /api/admin/staging`,
  `POST /api/admin/tasks/[id]/distribute`, page `/admin/staging`.
- Schema (additive): `Project.isStaging/asanaId`, `Task.asanaId/distributedAt/originalAssignee`,
  `Comment.asanaId`.

## Asana-like subtasks
- Subtasks are now **clickable to open** in their own panel (drill-down → nesting), with a
  **"← Back to parent task"** bar.
- Each subtask has an **inline assignee** and a clear **Delete**.
- **Drag-to-reorder** subtasks (replaces the up/down arrows).
- The task GET now returns `project.isStaging` so the panel can show staging controls.
- Admins can open any task (task-access checks now allow `role === "ADMIN"`).

## UI / UX overhaul
- **Design system**: `components/ui/Button.tsx` (primary/secondary/ghost/danger/subtle +
  sizes), `components/ui/icons.tsx` (crisp inline SVGs replacing emoji),
  `components/ui/Toast.tsx` (success/error toasts via a provider).
- **Drag-and-drop** everywhere via `@hello-pangea/dnd`: board cards reorder within a column
  **and** move across columns (fractional ordering, optimistic + persisted); subtasks reorder.
- **Board**: columns now fill the width, status-colored headers + counts, refined cards
  (priority dots, assignee avatars, due/subtask footer), empty-column states, and an
  **in-column "+ Add task"** composer (Enter to add, Esc to cancel).
- **Board search + assignee filter** (incl. Unassigned).
- **Global header search** (`GET /api/search` + `GlobalSearch`): finds projects **and** task
  titles (decrypted server-side, scoped to what you can access, staging excluded); results jump
  to the board with the task panel open. Plus a quick filter box on the Projects page.
- **Task panel**: wider (540px), sticky Save bar, dimmed **backdrop** (click to close) + **Esc**.
- **Clear deletes everywhere**: trash-icon + label buttons (subtasks, members, tasks, projects,
  staged tasks/projects) instead of faint ✕. `DELETE /api/projects/[id]` is now admin-only with
  proper errors; DB cascades verified.
- **Back navigation**: board pages show "← Back to Projects" / "← Back to Staging"; staged
  boards show a "📥 Staging" context banner.
- **Projects list**: modern cards with a done-% progress bar, status dots, 4-up grid.
- **Dashboard**: refreshed stat cards (icon tiles + accents).
- **Login**: premium split layout (brand panel + clean form, contextual heading).
- **Sidebar**: live project list + active-route highlighting + icons; "Staging (Asana import)"
  admin link.

## Data import (the actual Asana content)
- Imported all **7 active** Asana projects into Staging (~441 unique top-level tasks, ~430
  subtasks, real comments — Asana bot/reminder notifications filtered out). The 11 "DO NOT USE"
  archives + empty "Sidney" project were skipped.
- **Statuses** were derived from each task's **Asana section** (Asana has no To Do/In Progress
  status — only named sections): In Progress / Daily Task / On Hold → In Progress, For Review →
  In Review, Done/completed → Done, else To Do.
- Audited PT Sehat: **218/218 imported, nothing missing**. Fixed **57 multi-homed tasks** to take
  their status from their **own** board's section. See ASANA_STAGING.md → "Multi-homing".
- **Staging UI defaults to a Board view** (grouped by status) with a toggle to stacked Rows.
